## 컬렉션 프레임워크

> 자료구조: 자료를 저장할 수 있는 구조

### Collection 인터페이스
- 자료가 있다는 것을 표현
- 중복 허용, 저장 순서 기억 못 함

### Set
- **중복 비허용**, 순서 없음
- `Collection` 인터페이스 상속
- `add()` — 같은 자료 있으면 `false`, 없으면 `true` 반환
- 종류: `HashSet`, `TreeSet`

### List
- **중복 허용**, **순서 기억**
- `Collection` 인터페이스 상속
- `get(int)` — 특정 인덱스 자료 반환
- `size()` — 크기 반환
- 배열과 달리 저장공간이 필요에 따라 자동 확장

### Map
- **Key-Value** 쌍으로 저장
- `put(key, value)` — 저장
- `get(key)` — 값 조회
- `keySet()` — 모든 Key를 `Set`으로 반환
- Key는 중복 불가

## Generic

> Java 5에서 도입. 인스턴스 생성 시 사용할 타입을 지정하는 문법

- 선언 시: 가상의 타입 (예: `T`, `E`)
- 사용 시: 구체적인 타입 지정 (예: `<String>`, `<Integer>`)

## Set — Iterator 사용법

```java
Iterator<String> iter = set.iterator();

while (iter.hasNext()) {  // 데이터 있는 동안 true
    String str = iter.next();  // 값 꺼내기
    System.out.println(str);
}
```

## Queue — poll() 메서드

- `poll()` — Queue의 **헤드(최상위 요소)를 꺼내고 제거**
- 레벨 순회(BFS) 구현 시 주로 활용
